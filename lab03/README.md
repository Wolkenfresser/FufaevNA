# Лабораторная работа №03: Рекурсия

## Цель работы
Освоить принцип рекурсии, научиться анализировать рекурсивные алгоритмы и понимать механизм работы стека вызовов. Изучить типичные задачи, решаемые рекурсивно, и освоить технику мемоизации для оптимизации рекурсивных алгоритмов.

## Теоретическая часть
- **Рекурсия** – процесс, при котором функция вызывает саму себя для решения задачи.
- **Базовый случай** – условие, которое прекращает рекурсивные вызовы.
- **Рекурсивный шаг** – разбиение задачи на более простую подзадачу того же типа.
- **Глубина рекурсии** – количество вложенных вызовов функции.
- **Стек вызовов (Call Stack)** – хранит информацию о незавершённых вызовах функций.
- **Мемоизация** – техника оптимизации, сохраняющая результаты функций для повторного использования.

## Практическая часть

### Задания
1. Реализовать рекурсивные алгоритмы: факториал, числа Фибоначчи, быстрое возведение в степень.
2. Оптимизировать вычисление чисел Фибоначчи с помощью мемоизации и сравнить производительность.
3. Решить практические задачи:
   - бинарный поиск рекурсивно,
   - обход файловой системы,
   - "Ханойские башни" для n дисков.
4. Провести замеры времени и глубины рекурсии, визуализировать результаты.
